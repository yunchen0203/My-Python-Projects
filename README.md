## 🛠️ 目前專案內容

### ✒️ QuickSign.ipynb (重點專案)
* **功能描述**: 為了解決傳統紙本簽到效率低、易出錯問題而開發的自動化簽名處理工具。
* **語言**: Python 3
* **主要函式庫**: `pandas`, `openpyxl`
* **開發環境**: Jupyter Notebook / VS Code
* **進度**: 開發中
  
#### 🌟 核心功能
1. **自動化名單導入與初始化**：使用 pandas 讀取 .xlsx 或 .csv 格式的原始學生名單。
* **詳細描述**： 程式啟動後會自動掃描指定目錄下的 Excel 檔案，並將學生姓名、學號與班級資訊載入至 DataFrame 物件中，取代過去需要手動輸入名單的繁瑣步驟，確保資料來源的單一性與準確性。

2. **智慧型重複簽到攔截機制**：利用 Python 的 set 資料結構或 pandas 的 isin() 邏輯判斷。
* **詳細描述**：針對「輸入學號」進行即時比對。若該學號已存在於「已簽到列表」中，系統會立即彈出警告提示，有效防止因代簽或重複操作造成的統計誤差，大幅提升簽到資料的信賴度。

3. **異常輸入與防錯處理** ：結合 try-except 區塊捕捉 ValueError。
* **詳細描述**：當使用者輸入非數字字元或未定義的學號時，系統不會直接崩潰，而是透過異常處理機制引導使用者重新輸入。這對於在活動現場由不同人員操作時，能保持程式運行的穩定性。

4. 自動化格式化資料匯出：使用 openpyxl 進行 Excel 引擎串接與資料排序。

* **詳細描述**：簽到結束後，程式會自動依照「班級」或「學號」進行升冪排序，並生成帶有當前時間戳記（Timestamp）的新報表。這解決了傳統紙本簽到後，還需要人工二次輸入與排序的痛點。

#### 🧠 使用的程式邏輯
* **迴圈 (Loops)**: 用於逐行檢查名單資料與持續等待輸入。
* **條件判斷 (If-Else)**: 判斷學號是否存在於名單中。
* **例外處理 (Try-Except)**: 防止輸入錯誤導致程式中斷。
