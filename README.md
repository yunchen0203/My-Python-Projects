這是一個存放 Python 練習與開發紀錄的儲存庫。

## 🛠️ 目前專案內容

### ✒️ QuickSign.ipynb (重點專案)
* **功能描述**: 為了解決傳統紙本簽到效率低、易出錯問題而開發的自動化簽名處理工具。
* **語言**: Python 3
* **主要函式庫**: `pandas`, `openpyxl`
* **開發環境**: Jupyter Notebook / VS Code
* **進度**: 開發中

#### 📝 開發動機 (Motivation)
身為校級幹部，我觀察到傳統紙本簽到存在三大痛點：
1. **效率低**：活動現場常因多人排隊手寫資料而造成擁擠。
2. **資料易出錯**：成員抵達集合地點的時間分散，常有人因趕場而遺漏簽到，或在彩排結束後的混亂中忘記補簽。
3. **行政負擔**：所有集合彩排簽到紀錄皆須由幹部確認資料無誤後交給學校負責行政老師做後續處理，然而手動簽到排序混亂，除了幹部比對資料會耗費大量時間外，老師在處理資料時可能也會不方便。
  
#### 🌟 核心功能
1. **自動化名單導入**：利用 `pandas` 串接 `.xlsx/csv` 檔案，實現學生資訊自動初始化，取代人工錄入。
2. **重複簽到攔截**：透過邏輯判斷實作即時檢核，防止重複輸入同一學號，確保簽到數據的唯一性與信賴度。
3. **健全的異常處理**：內建 `try-except` 與格式檢核（如長度限制、類型檢查），能有效過濾輸入錯誤，維持系統穩定運行。
4. **格式化資料匯出**：整合 `openpyxl` 引擎，匯出前自動執行「班級座號排序」，產出可直接使用的標準化 Excel 報表。

#### 🧠 使用的程式邏輯
* **無窮迴圈 (`while True`)**：作為系統主引擎，持續等待使用者指令，直到輸入「結束」才觸發 `break` 跳出。
* **嵌套迴圈 (Nested Loops)**：在登錄與簽到功能中使用次級迴圈，確保使用者在輸入錯誤時能「原地修正」，直到資料正確才回到主選單。
* **條件判斷 (if-elif-else)**：實作多重檢核，精確判別學號是否已登錄、格式是否正確或重複簽到。
* **例外處理與格式攔截**：透過長度判斷與型別檢查，主動過濾非法輸入，確保系統運行不中斷。
* **資料管道 (Pandas)**：自動將記憶體中的數據排序並轉換，最終產出標準化 Excel 報表。
